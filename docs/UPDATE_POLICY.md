# 更新检测策略

本仓库是基于 `Wei-Shaw/sub2api` 的二次开发版本，应用内更新以当前仓库 `1459547510/sub2api-so-fy` 的 Release 为实际安装来源。

## 当前仓库

- 检测当前仓库最新 Release，作为一键更新的下载目标。
- 当前仓库 Release 高于运行版本时，允许应用内下载、替换二进制并重启。
- 当前仓库分支提交信息可作为源码构建参考，但不会替代 Release 更新包。

## 原仓库

- 原仓库只检测最新 Release 版本号。
- 原仓库 main 分支有新提交但没有新 Release 时，不提示“有新版本”。
- 原仓库 Release 版本高于当前版本且当前仓库 Release 尚未同步时，只提示需要先同步、合并并发布当前仓库 Release；应用内更新不会直接下载原仓库包。

这个策略避免因为原仓库普通提交触发误报，也避免应用内更新覆盖二次开发新增或修改的内容。
